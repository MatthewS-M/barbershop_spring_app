# **Введение**

Барбершопы – это уютные и стильные места, где мужчины могут получить профессиональный уход за своим внешним видом. Однако управление таким бизнесом может быть сложным заданием, особенно когда речь идет о множестве объектов, сотрудников и клиентов.

В свете этого актуальным становится разработка информационно-справочной системы для барбершопов, которая поможет улучшить учет и управление объектами предметной области, а также обеспечить быстрый доступ к справочной и статистической информации.

Цель данной курсовой работы заключается в разработке такой системы, используя SpringBootJava и двухуровневую структуру данных типа «родитель-дочка» в информационной модели предметной области. В добавок к этому будет применен метод Role-basedauthentication, который позволяет проводить регистрацию полизователей, сохранение их (credentials) в БД, а также их аутентификацию для получения доступа к клиенту. В зависимости от роли пользователя предусмотрены соответствующие для него права. Клиентская часть системы будет реализована с графическим интерфейсом при помощи HTML-разметки и будет обеспечивать соединение с сервером при помощи запросов к API. Серверная часть системы будет отвечать за обработку данных, полученных от клиента, сохранение их в БД, а также выполнение запросов к БД.

Ожидаемым результатом работы будет функциональная и эффективная информационно-справочная система, которая поможет упростить управление барбершопом, повысить качество предоставляемых услуг и удовлетворить потребности как владельцев, так и клиентов такого бизнеса.

# **Постановка задачи**

Для реализации информационно-справочной системы барбершопа необходимо разработать клиент-серверное приложение, используя библиотеку Spring Boot для серверной части и HTML (HyperTextMarkupLanguage) разметку с интеграцией некоторых библиотек и фреймворков для GUI (Graphical User Interface) клиентской части. База данных будет реализована на MySQL, с использованием ORM (Object-RelationalMapping) для связи с сервером. Решение должно выполнять следующие функции: отображение данных предметной области в таблице, добавление, редактирование и удаление объектов в БД, фильтрация и сортировка записей, отображение статистических данных. Кроме того, приложение должно обеспечивать удобный и интуитивно понятный пользовательский интерфейс, содержать комментарии, необходимые для автоматической генерации документации при помощи инструмента JavaDoc, и предотвращать аварийное завершение в случае ошибок ввода данных или других нештатных ситуаций. В результате работы должно быть создано полнофункциональное приложение, соответствующее требованиям поставленной задачи.

# **Описание предметной области.**

Предметная область данной курсовой работы – веб-приложение, которое будет выполнять учет и отображение объектов предметной области барбершопа, а именно его клиентов и рецензий, которые они составили. **Данные предметной области отображаются в таблице, все эти записи хранятся в базе данных.** Логика моей информационно-справочной системы барбершопа следующая: первый созданный пользователь – администратор. Он регистрируется на кастомной странице регистрации (/auth/register), указывая свой логин (username), пароль (password) и роль (role). После регистрации его credentials сохраняются в созданную, подвязанную к классу User\_info, таблицу user\_info, в которой указывается id, username, password, role каждого пользователя. Пароли шифруются в базе данных с помощью BCryptPasswordEncoder. Создавать новых пользователей может только пользователь с ролью ROLE\_ADMIN. То есть, учитывая реализацию role-basedauthentication, пользователю с ролью администратора предоставляется полный функционал работы с приложением. Администратор способен добавлять пользователей, изменять/удалять/добавлять записи в таблицы с клиентской части. Пользователь в свою очередь способен лишь читать информацию с сайта, смотреть статистические параметры.

**Ограничение прав доступа прописано тремя методами:**

1 – Права доступа ограничены на backend с помощью ".requestMatchers("/auth/\*\*").hasAuthoruty("ROLE\_ADMIN")" вметоде SecurityFilterChain. Позволяет предоставлять доступ ко всем страницам с путем "auth".

2 – Права доступа ограничены на backend в AppController с помощью аннотации "@PreAuthorize("hasAuthority('ROLE\_ADMIN')")" перед методом GetMapping, который предоставляет доступ к какой-либо странице.

3 – Права доступа ограничены на frontend с помощью Thymeleafexpressions, а именно их условных конструкций, которые проверяют, является ли пользователь в текущей сессии администратором или обычным пользователем. Например, "\<th:block th:if="${#authorization.expression('hasRole(''ROLE\_ADMIN'')')}"\>".
Тем самым, в зависимости от роли пользователя ему предоставляется определенный функционал.

# **Описание программы**

## **4.1.**  **Алгоритмические решения**

После отправления клиентом запроса на сервер, сервер его сопоставляет с одним из прописанных методов в контроллере в зависимости от того, какой используемых метод для запроса-ответа между клиентом и сервером был предпринят: GET - запрашивает данные из указанного ресурса; POST - отправляет данных, подлежащие обработке, на указанный ресурс. В дальнейшем в контроллере определенный метод либо сразу перенаправляет на определенную страницу (кастомную html, находящуюся в templates), либо обрабатывает считанные или переданные данные с этой странице и использует их на другой. Эти махинации проходят с помощью представления ModelAndView, которое позволяет добавлять некоторые объекты или атрибуты на страницу html для дальнейшего их использования. В добавок к этому в некоторых методах в контроллере происходит обработка данных из БД путем взаимодействия с репозиторием через сервис. Класс Service содержит бизнес-логику каждой из таблиц, используемых в приложении. Сервис обращается к своему репозиторию, каждый из репозиториев позволяет представляет собой хранилище для соответствующих объектов, которые сохраняются в реляционной базе данных. Администратору предоставляется возможность проводить манипуляции над записями БД, тем самым **обновляя изменения источника данных в базе данных.**

Этот интерфейс расширяет интерфейс JpaRepository, который предоставляет операции CRUD (создание, чтение, обновление, удаление) для объектов в базе данных. Он определяет метод поиска, который использует JPQL (Java Persistence Query Language) для поиска клиентов, соответствующих заданному ключевому слову. Этот интерфейс является важной частью Spring Data JPA framework, и он отвечает за обеспечение уровня абстракции между приложением и базой данных.
Клиент отправляет запросы на сервер согласно API, представленному в виде URL адресов в первой части взаимодействия клиента с сервером. Изначально (на этапе тестирования проекта) использовался Postman - приложение для работы с API, популярный API клиент, который позволяет разрабатывать, тестировать и документировать API. В дальнейшем после многократных попыток реализации корректного ответа на запросы были созданы html страницы, которые содержат в себе объекты и атрибуты. Они включают в себя объекты базы данных, извлеченные с помощью наших репозиториев, ключевые слова, по которым извлекалась определенная строка из таблицы (производится поиск по определенному критерию), имя и роль текущего пользователя в сессии. Все эти атрибуты и объекты подвергались итерации и отображению с помощью выражений Thymeleaf (библиотека в Java, шаблонизатор, используемый для анализа и преобразования данных на странице HTML, XML, JavaScript, CSS).

## **4.2. Описание интерфейса программы**

В данной курсовой работе GraphicalUserInterface был реализован с помощью HTML разметки. Стили к каждой странице html интегрированы в сам файл. Отдельный HTML-элемент \<style\> содержит стилевую информацию для документа или его части. По умолчанию стилевые инструкции внутри этого элемента считаются написанными на CSS. По большей части стили для элементов моих HTML-страниц прописаны в \<style\>, однако некоторые небольшие параметры, которые необходимы для определения стилей (правил) оформления документов, их дизайн, прописаны в стилях каждого элемента. Каждая страница HTML включает в себя несколько составляющих: \<head\> - содержит метаданные (данные о данных), \<body\> состоит из \<header\> и \<footer\> - в этой части находится весь контент, отображаемый пользователям. В добавок к этому на некоторый моих HTML-страницах есть элементы \<scripttype="text/javascript"\>. В них помещены функции на языке JavaScript, позволяющий проводить обработку, вывод, анализ данных со страницы. В дальнейшем они пригождаются для корректного вывода гистограмм, замены элементов, процессинга считанного текста. В добавок к этому довольно часто употребляется шаблонизатор Thymeleaf, упрощающий работу с объектами и атрибутами, переданными через ModelAndView в AppController. ## **4.3. Состав приложения**

### **4.3.1. База данных**

База данных построена на MySQL - реляционной системе управления базами данных (СУБД), которая распространяется как свободное программное обеспечение. Созданная схема "barbershop" состоит из 3 сущностей: user\_info, blog, client.

![](RackMultipart20231004-1-lkvbqo_html_68264a8af2c85d09.png) **Ключевые особенности:**

- Таблица client включает в себя следующие атрибуты: id, full\_name, master\_name, service, visit\_date, которые обозначают id клиента, его полное имя, имя обслуживающего барбера, тип предоставляемой услуги (название стрижки), дату посещения барбершопа соответственно. ### **4.3.2. Сервер**

Одним из ключевых компонентов серверной части приложения является SpringFramework, который предоставляет разнообразные классы и компоненты, упрощающие реализацию веб- и корпоративных приложений. Другим не менее важным серверным инструментом является Gradle – фреймворк для создания облачных и корпоративных систем, включает в себя множество модулей и плагинов для более эффективной работы с проектами. Gradle является альтернативой другому популярному инструменту для сборки проектов - Apache Maven. Однако, Gradle обладает более гибкой конфигурацией и управлением зависимостями, что делает его более привлекательным для некоторых проектов. Для взаимодействия с базой данных применяются инструменты ORM (Object-RelationalMapping), включающие в себя JPA (JavaPersistenceAPI) и Hibernate. JPA является спецификацией, описывающей правила взаимодействия с БД, а Hibernate, в свою очередь, реализует эти правила, облегчая работу с объектами базы данных без необходимости написания SQL-запросов в явном виде. Эти компоненты образуют основу серверной части приложения и позволяют реализовать необходимые функциональные возможности.

### **4.3.3. Клиент**

Компоненты клиентской части приложения, использующей веб-интерфейс, отображаемый в браузере в качестве клиента, включают в себя:

1. HTML- **разметка** : это основа любой веб-страницы. HTML (Hypertext Markup Language) является стандартным языком разметки веб-страниц и предоставляет возможность создания элементов страницы, таких как текст, изображения, ссылки, формы и т.д.
2. Для упрощения создания этой разметки используется **Thymeleaf** - шаблонизатор для веб-приложений, который позволяет легко создавать динамические HTML-страницы, включая директивы и атрибуты для управления отображением данных.
3. **CSS (Cascading Style Sheets):** этот язык используется для описания внешнего вида HTML-элементов на странице, включая шрифты, цвета, размещение элементов и т.д.
4. Кроме того, в клиентской части используются элементы **Bootstrap** - фреймворк для веб-разработки, который включает в себя CSS-стили и JavaScript-компоненты для создания адаптивных и мобильных веб-приложений. Это позволяет создавать пользовательский интерфейс, который будет выглядеть красиво и привлекательно на любом устройстве, а также легко управлять его расположением и поведением.
5. **JavaScript:** язык программирования, используемый для создания динамических веб-страниц. Он может использоваться для создания интерактивных элементов на странице, таких как выпадающие списки, анимации, а также для считывания текста со страницы, отображения элементов, динамического изменения их стилей.
6. **HTTP-протокол:** это протокол передачи данных, используемый в Интернете для обмена данными между клиентом и сервером. Он обеспечивает основу для взаимодействия между клиентом и сервером при обработке запросов на получение и отправку данных.

Все эти компоненты в совокупности обеспечивают работу веб-интерфейса, который используется в качестве клиента в данном приложении.

Этот интерфейс расширяет интерфейс JpaRepository, который предоставляет операции CRUD (создание, чтение, обновление, удаление) для объектов в базе данных. Он определяет метод поиска, который использует JPQL (Java Persistence Query Language) для поиска клиентов, соответствующих заданному ключевому слову. Этот интерфейс является важной частью Spring Data JPA framework, и он отвечает за обеспечение уровня абстракции между приложением и базой данных.

Он предоставляет методы поиска, создания, извлечения и удаления клиентов. @Service указывает, что этот класс принадлежит к уровню сервиса и может быть внедрен в качестве зависимости в другие компоненты. Класс предоставляет методы для перечисления всех клиентов, поиска клиентов по ключевому слову, сохранения, извлечения по идентификатору и удаления записей. Он зависит от интерфейса репозитория блогов и использует его методы для взаимодействия с базой данных.

Этот класс представляет собой важную часть SpringSecurityframework и отвечает за защищенность веб-приложения и обеспечение его защиты от несанкционированного доступа и других угроз безопасности.
